# 平台账号重置密码策略设计

## 功能范围

本功能属于平台权限域，入口为“系统管理 → 平台设置”。平台管理员重置平台账号密码时，服务端按照系统策略生成临时密码。

## 策略

- `RANDOM`：服务端使用安全随机数生成 16 位临时密码，重置接口仅在本次成功响应中返回明文。
- `FIXED`：管理员配置固定临时密码，数据库只保存 BCrypt 哈希；重置接口不返回明文，避免密码在接口和日志中泄露。
- 两种策略都会使目标账号历史登录态失效。

## 安全边界

- 策略配置和重置操作分别受平台安全配置、平台账号密码重置权限保护。
- 固定密码不能通过查询接口回显，操作审计只记录策略类型，不记录密码。
- 当前实现服务平台账号；商户账号策略需在商户权限域单独设计，不能复用平台全局配置。
