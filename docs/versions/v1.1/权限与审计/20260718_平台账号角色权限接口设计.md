# 平台账号、角色与权限接口设计

## 范围

本功能位于 V1.1“权限与审计补强”阶段，仅覆盖平台权限域：平台账号管理、平台角色管理和平台权限点只读查询。商户账号与商户角色不在本次范围内。

## 接口边界

- 平台账号：查询、新增、修改展示名称与角色、启用停用、重置密码。
- 平台角色：查询、新增、修改、启用停用、查询和配置权限。
- 平台权限：仅查询；权限码由系统迁移维护，接口不提供创建、修改或删除。
- 所有接口位于 `/api/platform/v1/**`，必须以平台身份登录并执行权限码校验。

## 安全规则

- 平台账号和角色只查询 `PLATFORM` 域且 `tenant_id` 为空的数据。
- 不允许停用当前登录账号；系统预置角色不允许修改、停用或重新配置权限。
- 分配角色和配置权限前，必须验证操作者自身拥有目标角色及目标权限的全部权限码，避免权限提升。
- 新增、编辑、启停、重置密码、角色权限变更必须写入操作审计；密码明文不写日志。
- 操作审计必须使用结构化审计事件：业务服务声明动作枚举和对象快照，审计组件统一补全操作者、租户、追踪号和请求上下文，并生成中文展示内容；当前与业务事务同步写入，后续有可靠异步投递需求时再评估事务后事件或 Outbox 模式。
- 角色、用户与关系主键使用数据库自增；角色增加 `status` 字段，确保启停状态由数据库保存。

## 系统预置角色

以下角色由数据库迁移维护，均为不可通过管理接口修改、停用或重新配置权限的系统预置角色：

| 角色 | 职责与权限 |
| --- | --- |
| 首席管理员 | 保留原 `super_admin` 角色主键和全部平台权限，避免既有账号角色关联失效。 |
| 平台运营 | 开通、查询和启用商户。 |
| 客户顾问 | 仅查询商户基础信息。 |
| 系统管理员 | 管理平台账号、角色和权限点，不处理商户日常运营。 |
| 财务审计 | 仅查询商户基础信息；平台财务明细接口尚未上线，届时再单独增加只读权限。 |

## 验收

- 超级管理员拥有本次平台管理权限，并可完成账号、角色和权限查询流程。
- 非平台域令牌不能访问平台接口；缺少权限码时返回统一无权限响应。
- 重复手机号、非法角色、跨权限分配、修改系统角色和停用本人均返回中文业务错误。
