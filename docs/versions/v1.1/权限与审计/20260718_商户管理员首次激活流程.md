# 商户管理员首次激活流程

## 范围

平台开通商户后，商户管理员通过邀请码设置首次登录密码。本功能仅补齐后台前端入口，不改变既有租户开通、邀请码生成和服务端激活规则。

## 用户流程

1. 平台管理员开通商户并将管理员邀请码提供给商户负责人。
2. 商户负责人访问 `/merchant/activate`，输入邀请码、设置密码并确认密码。
3. 前端校验密码至少 8 位且两次输入一致后，调用 `POST /api/admin/v1/auth/activate`。
4. 服务端校验邀请码未使用且未过期，创建管理员账号并将邀请码标记为已使用。
5. 激活成功后跳转至商户管理端登录页，使用开通时预留的手机号和新密码登录。

## 邀请码重发

- 平台租户管理中，管理员状态为“待激活”时可以重新生成邀请码。
- 重发会使该租户全部旧的未使用邀请码立即过期，新的明文邀请码只在本次响应中展示。
- 管理员已经激活后不能重发邀请码，避免生成重复管理员账号。
- 平台可停用商户；停用不会删除历史数据，但会阻止商户登录和后续业务请求。

## 安全边界

- 激活页面和激活接口不依赖已登录会话，但只接受一次性、带有效期的邀请码。
- 前端校验只用于改善交互；邀请码状态和密码规则仍由服务端最终校验。
- 页面不展示或持久化密码，也不在错误提示中暴露邀请码的内部状态。

## 验收记录

- 未登录用户可访问 `/merchant/activate`，不会被路由守卫跳转。
- 已登录用户访问该页面会回到所属管理端首页。
- 激活成功后自动进入预选“商户管理端”的登录页。
- 已执行 `npm run build`，TypeScript 检查与 Vite 生产构建通过。
