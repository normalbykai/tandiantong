# 商户端基础设施页面设计与验收说明

## 设计范围

本次完善 V1.1 商户管理端的基础设施页面，沿用现有“青墨”后台排版、卡片、表格、标签和商户域墨青色变量。页面只服务租户权限域，不复用平台管理端的数据、角色或权限配置。

当前包含以下页面：

| 页面 | 路径 | 当前能力 |
| --- | --- | --- |
| 工作台 | `/merchant/dashboard` | 展示基础设施快捷入口和单门店范围说明 |
| 门店信息 | `/merchant/store` | 查看单门店资料、营业安排，提供编辑交互入口 |
| 员工账号 | `/merchant/staff` | 搜索、状态筛选和查看当前商户员工账号 |
| 商户角色 | `/merchant/roles` | 查看当前商户角色，进入权限明细入口 |
| 商户权限 | `/merchant/permissions` | 按名称、编码和模块查询商户权限点 |
| 操作日志 | `/merchant/logs` | 按操作人、对象、追踪号和操作类型查询商户日志 |
| 系统管理 | `/merchant/system` | 维护商户展示偏好，查看商户账号安全和数据范围 |

## 权限域边界

- 商户页面路由统一标记为 `TENANT`，平台登录态不能进入商户页面。
- 商户角色只能配置当前商户范围内的权限，不能授予平台账号、平台角色、租户管理和平台系统权限。
- 单门店是 V1.1 当前边界，页面不提供新增门店、门店切换或多门店数据权限配置。
- 页面上的菜单、按钮和筛选只改善操作体验，真实接口接入后仍必须由后端执行租户、门店和权限校验。

## 交互要求

- 门店资料编辑、角色权限查看和接口加载失败必须使用明确的中文结果反馈。
- 列表页面保持筛选、状态标签、空数据承载和表格操作区，不使用无界查询。
- 员工、门店和角色测试数据均为虚构数据，不包含生产账号和真实敏感信息。
- 门店、员工、角色、权限和操作日志页面已接入商户应用服务接口，不读取平台管理端数据。
