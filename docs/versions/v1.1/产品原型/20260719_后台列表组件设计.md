# V1.1 后台列表组件设计

## 设计目标

统一商户后台和平台管理列表的展示密度、文字层级和交互反馈。组件基于 Element Plus `el-table` 二次封装，页面只负责列定义和业务操作，不重复维护表格外观。

## 视觉约定

- 表头使用浅灰背景、12px 字号和中等字重，保证字段标题清晰可扫描。
- 普通单元格使用 13px 字号，行高约 54px；需要更高密度时使用 `density="compact"`。
- 主字段、状态标签和操作按钮保持明显层级，金额和编码等技术字段按页面需要使用等宽字体。
- 鼠标悬停时使用当前权限域的浅色渐变高亮整行，固定操作列保持可见。
- 列表卡片保留圆角、细边框和轻阴影，与“墨璃”后台视觉保持一致。

## 功能约定

- 通过 `loading` 统一显示加载状态，通过 `emptyText` 统一配置中文空状态。
- 通过 `rowKey` 为列表行提供稳定标识，避免刷新或状态变化时产生错误复用。
- 通过 `toolbar` 和 `footer` 插槽承载页面筛选、统计或扩展操作。
- 需要服务端分页时启用 `showPagination`，由 `page-change` 和 `page-size-change` 事件驱动数据请求；不允许使用无界查询。
- 业务按钮的权限、二次确认和幂等仍由页面与后端负责，列表组件只提供展示容器和分页事件。

## 当前接入范围

首个接入页面为“租户管理”，后续平台账号、角色、权限点和系统字典列表按页面改造时逐步迁移，不夹带业务接口变更。
