# V1.1 列表页面布局设计

## 页面结构

后台数据列表页面统一采用五段式卡片布局，各段之间保持 16px 间距，避免标题、操作和数据内容堆叠在同一张卡片中。

1. 标题栏：展示页面标题、模块描述和页面级入口。
2. 数据栏：展示当前模块的核心统计数据。
3. 功能栏：集中放置刷新、导入、导出和批量操作等通用功能；没有实际能力时不展示占位按钮。
4. 筛选栏：放置关键词搜索、状态筛选和其它查询条件。
5. 数据列表栏：只承载表格、空状态、加载状态和分页内容。

## 视觉规则

- 每段使用独立卡片、统一圆角和轻边框，卡片之间使用固定间距分隔。
- 标题栏强调页面身份，数据栏强调数字层级，功能栏和筛选栏使用紧凑布局，列表栏保持完整横向空间。
- 移动端将标题操作和筛选条件改为纵向排列，统计数据按两列或单列排列。

## 当前实施

租户管理页面作为首个实施页面，使用 `ListPageLayout` 统一承载五段布局；后续列表页面按相同插槽结构逐步迁移。
