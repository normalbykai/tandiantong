# V1.1 界面与交互基线

## 总体原则

- 所有用户可见文案使用简体中文。
- 商户后台以高频运营效率为目标，采用紧凑、清晰、可扫描的信息布局。
- 小程序优先保证扫码进入、商品购买、预约和凭证查看流程顺畅。
- 手机浏览器必须支持商户快速核销，关键操作提供明确成功、失败和重复状态。

## 商户后台

- 导航按商品库存、订单、预约、核销、经营数据和权限设置组织。
- 列表提供明确状态、筛选、分页和空状态，不使用无界查询。
- 金额界面显示元，但提交和后端处理统一使用分。
- 退款、核销和权限变更必须二次确认并显示可理解的结果。

## 顾客小程序

- 入口码失效、商品下架、库存不足和预约满额必须使用中文业务提示。
- 提交按钮防止重复点击，但后端幂等仍是最终安全边界。
- 取餐号只用于展示识别，核销必须使用不可猜测凭证。
- 支付结果以服务端查询或回调确认状态为准，不根据前端跳转直接判定成功。

## 原型继承

V1 的视觉、顾客端和后台原型只作为历史参考。V1.1 新增或修改页面时，应在本目录新增独立中文设计文档，不直接修改已冻结的 V1 原型。
