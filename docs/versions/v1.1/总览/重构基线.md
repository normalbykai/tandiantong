# V1.1 重构基线

## 已完成调整

- 全部数据库业务访问从 `JdbcTemplate` 迁移到 MyBatis-Plus。
- SQL 只允许位于 Mapper 注解或映射文件，应用服务不保存 SQL 字符串。
- 数据库实体使用普通类、表映射注解和 Lombok；请求、响应、命令和值对象继续使用 record。
- 自研 JWT 过滤器与令牌服务已删除，登录态统一交给 Sa-Token。
- 平台与租户使用独立权限域，租户和门店由服务端可信上下文建立。
- 库存锁定、支付确认、退款回补、预约容量和核销状态使用原子更新。
- 核销模块通过业务完成处理器通知订单模块，不直接访问订单持久层。
- Knife4j 已加入后台和小程序接口，接口分组与摘要使用中文。
- 生产 Java 类和数据库实体字段已补充中文 JavaDoc。
- Flyway 已增加数据库表中文含义迁移。

## 代码扫描基线

后续提交必须保持以下结果：

- 生产代码和 Maven 模块中不存在 `JdbcTemplate`。
- 应用服务中不存在查询、插入、更新或删除 SQL 字符串。
- 实体类不手写普通 getter 和 setter。
- 生产 Java 文件必须包含中文类级 JavaDoc。
- 敏感 Mapper 必须显式携带租户和门店条件。

## 已知限制

- 当前微信支付适配器仍是本地开发实现，未代表真实微信支付接入完成。
- 预支付和退款仍是同步流程，真实适配接入时必须评估事务拆分与失败补偿。
- 经营事实表尚无独立同步链路，当前看板直接读取交易事实表。
- 数据库已补充表级中文注释；字段级含义以实体字段 JavaDoc 为当前权威来源。
