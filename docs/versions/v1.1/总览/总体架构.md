# 摊点通 V1.1 总体架构

## 系统定位

摊点通是面向线下零售与到店服务商户的多租户系统。当前以单租户单门店交付，包含顾客微信小程序、商户响应式后台和平台管理能力。业务表保留门店字段，但 V1.1 不实现多门店运营。

## 模块结构

后端采用 Maven 多模块的模块化单体，所有模块最终装配为一个 Spring Boot 应用。

```text
tandiantong-bootstrap      启动、配置和统一装配
tandiantong-framework      可复用框架组件、自动配置和基础设施 starter
tandiantong-common         旧包名兼容层，新代码使用 tandiantong-framework-common
tandiantong-security       登录、用户、租户上下文和权限
tandiantong-catalog        商品、SKU、加料和库存
tandiantong-order          商品订单、支付和退款
tandiantong-reservation    服务项目、时段、容量和预约
tandiantong-verification   取餐号、核销凭证和核销记录
tandiantong-analytics      经营指标和导出审计
tandiantong-integration    微信、支付和对象存储适配
tandiantong-admin-api      平台与商户后台接口
tandiantong-mini-api       顾客小程序接口
```

接口模块只处理协议适配、输入校验和响应转换。业务模块之间通过应用服务或明确接口协作，禁止跨模块调用 Mapper。第三方调用集中在集成模块。

## 分层规范

- 控制层：接收请求、校验参数、调用应用服务和转换响应，不编写业务规则。
- 应用层：组织业务流程、事务、幂等和模块协作，不编写 SQL。
- 领域层：保存状态机、业务值和不依赖数据库的规则。
- 实体层：使用普通类映射数据库表，使用 Lombok 减少访问器代码。
- 持久层：Mapper 承担查询、聚合、行锁和原子条件更新。

## 多租户安全

- 后台租户和门店来自认证后的服务端上下文。
- 小程序入口码由服务端解析为可信租户和门店。
- 支付回调通过平台业务单号反查租户，不依赖登录态或客户端租户参数。
- MyBatis-Plus 租户拦截器提供基础隔离；订单、退款、预约、核销和权限操作必须再次校验资源归属。
- 平台权限域与租户权限域独立，不允许伪造租户身份。

## 交易安全

- 金额使用整数分，不使用浮点数。
- 创建订单、支付回调、退款、预约和核销必须幂等。
- 库存、预约容量和核销使用数据库原子条件更新。
- 支付、退款和核销状态必须遵守合法状态机。
- 订单、支付、退款、库存流水、预约、核销和审计记录不得物理删除。

## V1.1 范围边界

V1.1 不包含多门店、配送、套餐、人员排班、动态数据权限、平台分账、搜索引擎、消息队列、微服务和容器编排平台。新增上述范围前必须单独更新设计并获得确认。
