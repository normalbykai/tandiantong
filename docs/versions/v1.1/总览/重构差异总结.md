# V1.1 重构差异总结

## 1. 重构目标

本次重构形成 V1.1 模块化单体后端基线，目标是移除业务服务中的 `JdbcTemplate` 和手写 SQL，统一使用 MyBatis-Plus 持久层；使用 Sa-Token 承担登录态与权限域认证；通过 Knife4j 提供可调试接口文档；使用 Lombok 简化数据库实体，并补齐中文 JavaDoc、实体字段含义和数据库表含义。

## 2. 重构前后差异

| 关注点 | 重构前 | 重构后 |
| --- | --- | --- |
| 持久层 | Service 注入 `JdbcTemplate` 并拼写 SQL | Service 注入 Mapper 或跨模块应用服务，SQL 仅位于 Mapper |
| 实体模型 | 持久化模型不完整，部分场景直接映射查询结果 | 每张业务表建立普通 class 实体，使用 `@TableName`、`@TableId` 和 Lombok |
| record 使用 | record 同时承担请求、结果和部分业务数据 | record 保留用于不可变命令、请求、响应和领域值；数据库实体统一使用 class |
| 认证 | 自研 JWT 令牌解析与过滤器 | Sa-Token 登录态、权限域登录标识和统一拦截器 |
| 接口文档 | 仅有普通 Spring MVC 接口 | 引入 Knife4j/OpenAPI，控制器增加中文 `@Tag` 和 `@Operation` |
| 租户隔离 | SQL 中分散编写租户条件 | MyBatis-Plus 租户拦截器作为基础隔离，敏感 Mapper 仍显式校验租户和门店 |
| 并发控制 | Service 中直接执行条件 SQL | Mapper 提供库存、预约容量、订单状态和核销状态原子更新方法 |
| 初始化 | 启动模块直接使用 `JdbcTemplate` 创建平台用户 | 启动配置调用安全模块应用服务，由安全模块 Mapper 持久化 |
| 注释 | 类头和字段中文说明不完整 | 生产 Java 文件均包含中文 JavaDoc；数据库实体字段逐字段标注中文含义 |

## 3. 阶段完成情况

### 阶段一：技术基线与架构设计

- 引入 MyBatis-Plus、Sa-Token、Knife4j 和 Lombok。
- 更新架构认证选型并新增专项重构计划。
- 增加 Mapper 扫描、分页插件和多租户拦截器。

### 阶段二：认证与租户基础设施

- 删除旧 JWT 过滤器、令牌服务和令牌声明对象。
- 数据库认证改为 Mapper 查询，并使用 Sa-Token 创建和解析登录态。
- 平台权限域与租户权限域使用独立登录标识。
- 新增入口码应用服务，将外部 `scene` 解析为可信租户和门店范围。

### 阶段三：安全与商品持久化

- 商户开通、用户、角色、邀请、支付配置和入口码改用 MyBatis-Plus。
- 商品、SKU、加料和库存流水改用实体与 Mapper。
- 新增库存应用服务，对订单模块提供定价、库存锁定、支付确认扣减和退款回补能力。

### 阶段四：订单与支付

- 订单主表、明细、支付、退款和幂等记录统一使用 MyBatis-Plus 实体。
- 支付回调按全局订单号反查可信租户，不依赖客户端租户参数。
- 订单支付、退款和完成状态使用带当前状态条件的原子更新。
- 下单和退款保留租户维度幂等记录；退款成功后回补库存并取消凭证。

### 阶段五：预约与核销

- 服务项目、时段、预约和预约幂等记录改用实体与 Mapper。
- 预约容量使用条件更新占用，取消使用行锁和条件更新释放容量。
- 取餐号序列使用唯一键和原子递增，避免并发重复。
- 核销凭证只保存 SHA-256 哈希，明文只在首次签发时返回。
- 核销通过业务完成处理器通知订单模块，不跨模块访问订单 Mapper。

### 阶段六：统计、接口文档和数据库说明

- 聚合 SQL 移入统计 Mapper，导出审计使用实体插入。
- 控制器增加 Knife4j/OpenAPI 中文分组与接口说明，请求 record 可直接用于接口调试。
- 新增 Flyway `V1_9_0`，为全部现有业务表写入中文表含义。
- 数据库字段中文含义与实体字段一一对应，写在各模块 `entity` 包字段 JavaDoc 中。

### 阶段七：规范与验证

- 清理实体手写 getter/setter，统一由 Lombok 生成。
- 生产 Java 文件完成类级中文 JavaDoc 覆盖。
- 增加库存原子锁定、预约容量不足和核销重复提交测试。
- 使用 JDK 21 完成编译和测试。

## 4. 当前包职责

- `controller` 或 API 模块：协议适配、参数校验、响应转换和 OpenAPI 描述。
- `app`：应用服务、事务边界、幂等编排和跨模块应用接口协作。
- `domain`：不依赖数据库的业务状态、值对象和领域规则。
- `entity`：数据库表映射，使用普通 class 和 Lombok。
- `mapper`：MyBatis-Plus `BaseMapper`、复杂查询和原子条件更新 SQL。
- `config`：框架、租户拦截器、认证和第三方适配器装配。

## 6. 明确限制

- 当前微信支付实现是开发替身，正式接入前仍需在 `tandiantong-integration` 完成证书验签、超时、重试和真实错误分类。
- 创建预支付和退款仍是同步流程。正式适配上线前，应结合真实接口耗时评估事务拆分、失败补偿和退款异步确认。
- 经营事实表尚未建立同步链路；当前看板直接读取交易事实表，符合第一阶段 YAGNI 范围。
- Flyway 新迁移补充了表级中文注释；字段级中文含义当前以实体字段 JavaDoc 为权威来源。生产数据库若要求字段 `COMMENT` 元数据，必须基于实际 MySQL 类型新增独立迁移，不能修改历史脚本。
