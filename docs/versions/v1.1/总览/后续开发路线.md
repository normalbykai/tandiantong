# V1.1 后续开发路线

## 第一阶段：权限与审计补强

- 从数据库角色与权限关系加载权限码。
- 对商品、退款、核销、导出和角色管理接口执行权限校验。
- 增加退款、核销、角色和权限变化的业务审计记录。
- 使用两个租户补充权限绕过和资源归属测试。

## 第二阶段：订单完整闭环

- 完成加料规则校验、价格计算和成交快照。
- 增加待支付订单超时取消与库存释放。
- 增加顾客订单列表、详情和允许范围内的取消能力。
- 完成退款失败记录、重试上限和人工排查状态。

## 第三阶段：预约完整闭环

- 完成付费预约预支付、回调确认和凭证签发。
- 完成预约超时取消与容量释放。
- 增加预约核销和履约状态原子更新。
- 增加顾客预约列表、详情和取消接口。

## 第四阶段：外部服务

- 接入微信小程序登录并绑定顾客身份。
- 接入真实微信支付证书、密钥、验签和退款查询。
- 按已确认需求接入对象存储和短信，不提交真实凭据。
- 覆盖超时、验签失败、重复回调和重试场景。

## 第五阶段：联调与发布验收

- 完成双租户、库存并发、预约并发和重复核销集成测试。
- 完成前后端关键流程联调和人工验收。
- 完成容器部署、生产配置、健康检查和数据库备份恢复验证。
- 汇总发布限制和回滚方案后再形成正式版本提交。
