# V1.1 测试与验收基线

## 自动化测试

- 金额、状态机、角色权限和租户上下文必须有单元测试。
- 库存、预约容量、支付、退款和核销必须有数据库集成测试。
- 租户隔离必须至少使用两个租户覆盖查询、更新和导出。
- 幂等测试必须验证重复请求不会重复扣减、回补、占用或核销。
- 第三方适配必须覆盖正常、超时、验签失败、重复回调和重试上限。

## 提交前命令

```powershell
$env:JAVA_HOME='D:\DevTools\jdk-21.0.11'
$env:PATH="$env:JAVA_HOME\bin;$env:PATH"
mvn -q -DskipTests compile
mvn -q test
git diff --check
```

## 当前验证结果

V1.1 持久层与认证重构已使用 Java 21 完成全量编译和全量测试。库存原子锁定、预约容量不足和核销重复提交已有新增测试。

## 验收门槛

- 编译、测试和差异检查全部通过。
- 不存在 `JdbcTemplate` 和应用服务 SQL。
- 新增业务规则有自动化测试。
- 数据库变更只通过新增 Flyway 迁移完成。
- 敏感操作完成租户归属、权限、幂等和审计检查。
- 未验证内容和外部依赖必须在交付说明中明确列出。
