## 规则说明

V1 历史迁移链已经归档，不再作为当前 V1.1 的 Flyway 执行链路。V1.1 从 `2.20260714.1` 建立数据库结构基线，新库直接执行完整初始化脚本，旧库保留数据时通过 Flyway baseline 标记到该版本。

Flyway 文件名必须保留 `V` 前缀，并使用两个下划线分隔版本号和描述。当前项目约定 V1.1 数据库迁移使用 `V2_日期_递增值__描述.sql`，例如 `V2_20260714_1__V1_1数据库完整结构基线.sql`。其中 `V2` 表示归档 V1 历史链后的第二条主迁移链，日期表示脚本创建日期，递增值表示当前主迁移链内的顺序。

`tandiantong-bootstrap/src/main/resources/db/migration/` 只保存当前要执行的迁移脚本。历史脚本必须放在不会被 `spring.flyway.locations` 扫描的位置，禁止把旧脚本继续放在迁移目录中。

## 新库初始化

新建数据库或可以清空重建的开发测试库，不启用 baseline，直接执行迁移：

```powershell
mvn -pl tandiantong-bootstrap -am spring-boot:run
```

也可以使用项目约定的启动方式触发 Flyway。此时 Flyway 会执行：

```text
V2_20260714_1__V1_1数据库完整结构基线.sql
```

## 旧库保留数据切换

旧库已经执行过 V1 历史迁移且需要保留业务数据时，必须按以下顺序处理：

1. 停止应用，避免迁移过程中产生新写入。
2. 完整备份数据库。
3. 确认当前库结构与 `V2_20260714_1__V1_1数据库完整结构基线.sql` 等价。
4. 备份 Flyway 历史表：

```sql
CREATE TABLE flyway_schema_history_v1_archive AS
SELECT * FROM flyway_schema_history;
```

5. 删除 Flyway 历史表：

```sql
DROP TABLE flyway_schema_history;
```

6. 只在本次切换时启用 baseline，并显式指定本次基线版本：

```powershell
$env:TDT_FLYWAY_BASELINE_ON_MIGRATE='true'
$env:SPRING_FLYWAY_BASELINE_VERSION='2.20260714.1'
$env:SPRING_FLYWAY_BASELINE_DESCRIPTION='20260714_V1.1_数据库结构基线'
```

7. 启动应用或执行迁移，Flyway 会把当前库标记为 `2.20260714.1`，不会重复执行完整初始化脚本。
8. 切换完成后移除 `TDT_FLYWAY_BASELINE_ON_MIGRATE`、`SPRING_FLYWAY_BASELINE_VERSION` 和 `SPRING_FLYWAY_BASELINE_DESCRIPTION`，恢复默认关闭状态。

也可以使用启动参数临时指定：

```powershell
mvn -pl tandiantong-bootstrap -am spring-boot:run `
  "-Dspring-boot.run.arguments=--spring.flyway.baseline-on-migrate=true --spring.flyway.baseline-version=2.20260714.1 --spring.flyway.baseline-description=20260714_V1.1_数据库结构基线"
```

## 后续变更

后续数据库变更必须新增更高版本迁移，禁止修改 `V2_20260714_1__V1_1数据库完整结构基线.sql` 和归档的 V1 历史脚本。

命名建议：

```text
V2_20260714_2__xxx.sql
V2_20260715_3__xxx.sql
```

版本号中的日期表示迁移创建日期，最后一段数字表示 V2 主迁移链内的递增序号。脚本说明必须清楚描述业务目的，禁止使用无意义名称。
