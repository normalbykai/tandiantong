# V1.1 版本迁移说明

## 迁移原则

V1 文档保留历史，不整体复制。只有与当前代码一致、仍能指导后续开发的内容才被总结后进入 V1.1，已经过时的实现细节不再继承。

## 有效内容迁移

| V1 资料 | V1.1 去向 | 处理方式 |
| --- | --- | --- |
| 总体架构 | `总览/总体架构.md` | 保留模块化单体、业务边界和多租户原则，更新技术基线 |
| 分层实施路线 | `总览/后续开发路线.md` | 删除已完成阶段，保留真实未完成能力 |
| 持久层与认证重构资料 | `总览/重构基线.md` | 合并为当前代码约束和已知限制 |
| 联调与测试现状 | `质量保障/测试与验收基线.md` | 删除旧分支和旧 JWT 描述，保留测试缺口 |
| 产品原型设计 | `产品原型/界面与交互基线.md` | 提炼仍有效的中文界面和交互原则 |

## 不再继承的内容

- 自研 JWT、`JdbcTemplate` 和 Service 手写 SQL 的实现说明。
- 指向旧开发分支、旧提交编号和过期测试日期的状态描述。
- 已完成重构阶段的逐文件实施步骤。
- 与当前数据库实体、模块边界或认证方式不一致的示例。

## 后续要求

从本版本开始，需求分析、设计、实施计划、测试记录和验收结论都必须写入 `docs/versions/v1.1/` 的中文目录和中文文件中。V1 目录只允许修正历史索引或弃用链接，不得继续承载开发内容。
