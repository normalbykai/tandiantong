# 摊点通 V1 文档索引

## V1 目标

V1 面向 10 至 100 个租户，交付 C 端微信小程序和统一 B 端响应式后台。范围包含商户开通、多租户 RBAC、商品与库存、订单与微信支付、服务预约、核销和基础经营数据。

V1 不包含配送、套餐、人员排班、动态数据权限、平台分账、自由页面装修和微服务基础设施。

## 状态说明

- `规划中`：正在澄清范围或编写设计。
- `待实施`：设计已确认，尚未开始编码。
- `实施中`：正在按功能单元开发和提交。
- `待验收`：功能完成，正在整体测试和用户确认。
- `已完成`：验收通过并完成合并。

阶段开始后按需创建以下文档：

```text
阶段目录/
├─ design.md                 阶段设计与业务边界
├─ prototype.md              仅在阶段包含页面交互时创建
├─ implementation-plan.md    可执行任务和功能单元提交计划
└─ acceptance.md             测试范围、验收标准和结果
```

## 实施顺序

每个阶段遵循以下流程：

```text
设计确认
→ 原型确认（如适用）
→ 实施计划
→ 按可独立验证的功能单元开发和提交
→ 阶段整体测试与联调
→ 用户验收
→ 进入下一阶段
```

禁止在一个实施计划中同时实现多个未确认阶段，也禁止为未来阶段提前创建空壳模块。
