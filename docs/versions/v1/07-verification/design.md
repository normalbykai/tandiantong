# 07 取餐号与核销设计

## 1. 阶段目标

本阶段为商品订单和服务预约生成不可猜测的核销凭证，并提供商户后台扫码、取餐号查询和确认核销能力。取餐号只用于现场识别，不能作为安全凭据。

## 2. 范围

- 商品取餐号按租户、门店和营业日生成。
- 商品订单与服务预约核销凭证。
- 核销记录和重复核销结果。
- B 端移动核销中心和核销详情页面。
- C 端取餐凭证与预约凭证安全提示。

## 3. 领域规则

- 核销令牌必须不可猜测，二维码展示令牌或其安全封装。
- 取餐号不能作为核销凭据，只能辅助查询。
- 核销必须校验租户、门店、业务类型、业务编号和当前待核销状态。
- 商品和预约核销必须执行带当前状态条件的原子更新或等价操作，重复或并发核销最多成功一次。
- 核销成功、重复核销和失败原因必须保留记录，日志和响应不得暴露完整敏感令牌。
