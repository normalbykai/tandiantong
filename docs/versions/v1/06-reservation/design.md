# 06 服务预约与容量控制设计

## 1. 阶段目标

本阶段交付服务预约能力。商户可以配置服务项目、价格和可预约时段；顾客选择日期时段后创建预约。免费预约直接确认并生成凭证，付费预约先进入待支付，支付成功后确认并生成凭证。

## 2. 范围

- 服务项目、免费或付费规则。
- 日期、时段、容量、暂停预约和剩余名额。
- 免费预约确认、付费预约待支付和支付后确认。
- 预约取消、容量释放和状态日志。
- B 端服务项目、预约列表和预约日历页面原型。
- C 端服务详情、日期时段选择、确认预约和预约凭证页面原型。

## 3. 不在本阶段实现

- 人员排班、房间、设备或跨门店资源预约。
- 复杂改期、候补、多人同行和周期订阅。
- 时段开始后的在线取消和退款。

## 4. 领域规则

- 预约容量必须通过条件更新、版本控制或等价原子方式占用，禁止先查后改导致超额预约。
- 免费预约确认后立即生成凭证。
- 付费预约创建后占用容量并进入待支付，支付成功后生成凭证；支付超时或取消释放容量。
- 已履约预约不得通过普通接口回退状态。
- 预约请求中的租户和门店来自服务端可信上下文，禁止信任客户端提交值。
