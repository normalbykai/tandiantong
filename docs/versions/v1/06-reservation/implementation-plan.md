# 06 服务预约与容量控制实施计划

## 1. 功能单元

1. 数据库迁移：创建服务项目、服务时段、预约记录和状态日志表。
2. 预约领域服务：创建服务、发布时段、免费预约确认、付费预约待支付、支付确认和取消释放容量。
3. 容量测试：覆盖最后一个名额、重复预约、租户隔离和取消释放。
4. 前端原型：B 端服务与预约页、C 端服务预约和预约凭证。
5. 验收验证：后端测试、前端审计、类型检查、构建和空白检查。

## 2. 验证方式

- `mvn -pl tandiantong-reservation -am test`
- `mvn test`
- `npm audit --audit-level=moderate`
- `npm run typecheck`
- `npm run build`
- `git diff --check`

## 3. 风险控制

- 不为未来人员排班、房间设备和多门店资源创建空壳模块。
- 容量变更集中在预约应用服务，后续数据库实现使用带剩余名额条件的原子更新。
