# 09 联调与验收设计

## 1. 阶段目标

09 阶段用于收束 V1 的工程、业务、前端、迁移、部署和验收证据。该阶段不新增业务范围，重点确认 01 至 08 阶段形成的能力可以共同构成 V1 交付基线。

## 2. 联调范围

- 平台创建商户、初始化门店、管理员邀请和启停用。
- 商户商品、SKU、加料、库存调整和库存流水。
- C 端商品下单、支付模拟、支付回调、退款和取餐凭证。
- 服务预约、容量占用、免费预约、付费预约模拟和预约凭证。
- B 端核销中心、重复核销展示和经营数据。
- 多租户隔离、RBAC、金额整数分、幂等、敏感信息和审计记录。

## 3. 部署边界

本地开发使用 Docker Compose 启动 MySQL 8 和 Redis。真实环境部署应当使用独立 MySQL、Redis、HTTPS 入口和安全配置注入。微信支付、微信小程序、短信和对象存储均需要真实凭据后才能完成生产级联调；当前只保留适配接口和本地模拟实现。

## 4. 验收原则

- 自动化测试通过后才记录阶段通过。
- 未接入真实凭据的项目必须列入待验证清单，不得标记为真实接入成功。
- 若后续接入真实外部服务导致业务边界变化，必须先更新对应阶段设计文档。
