# 08 经营数据实施计划

## 1. 功能单元

1. 经营数据服务：新增交易、商品、预约事实模型和聚合服务，覆盖租户隔离、金额统计、排行和时段使用率测试。
2. 导出审计：新增导出任务模型，校验独立导出权限、脱敏操作人和中文审计说明。
3. 数据库迁移：创建经营数据事实表和导出任务表，所有业务唯一索引和查询索引包含 `tenant_id`、`store_id`。
4. B 端页面：商户权限域新增“经营数据”菜单和页面，展示交易概览、商品排行、预约分析、导出任务与导出权限状态。
5. 阶段验收：运行经营数据模块测试、后端全量测试、B/C 端审计、类型检查、B 端构建和 `git diff --check`。

## 2. 验证重点

- 至少两个租户数据互不串扰。
- 退款金额不计入净收入，金额单位保持“分”。
- 导出任务记录脱敏联系人和中文审计说明。
- 前端仅在商户权限域展示经营数据入口。
