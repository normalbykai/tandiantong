# 04 商品、SKU、加料与库存验收

## 1. 验收范围

- 商品库存数据库迁移。
- 商品、SKU、加料和库存应用服务。
- B 端和 C 端商品库存页面原型。
- 库存流水和租户隔离测试。

## 2. 验收标准

| 项目 | 标准 | 结果 |
| --- | --- | --- |
| SKU 组合 | 同一商品内规格组合唯一，价格使用整数分 | 通过 |
| 加料规则 | 必选、最小和最大选择数量可校验 | 通过 |
| 上架限制 | 支付未验证时禁止上架付费商品 | 通过 |
| 库存流水 | 初始库存、调整、锁定、释放、扣减和回补均产生流水 | 通过 |
| 租户隔离 | 非当前租户或门店不能修改 SKU 库存 | 通过 |
| 构建测试 | 后端测试、前端类型检查和构建通过 | 通过 |

## 4. 已知限制

- 本阶段仅提供订单库存接口，不实现订单状态机和支付回调。
- 商品图片上传接入对象存储需要真实凭据，当前仅保留图片 URL 字段和接入边界。
- C 端 uni-app 工具链通过 overrides 固定 Babel、esbuild、ws、postcss、jpeg-js、phin 和 `@intlify/core-base` 的安全版本，后续升级 uni-app 时需要重新执行审计。
