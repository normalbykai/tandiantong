# 04 商品、SKU、加料与库存设计

## 1. 阶段目标

本阶段建立摊点通 V1 商品售卖基础。商户可以维护分类、商品、SKU、加料分组和库存，系统为后续订单创建提供可信商品快照、价格校验和库存锁定能力。

## 2. 范围

- 商品分类、商品基础信息、上下架状态。
- SKU 规格组合、编码、价格、库存和预警值。
- 加料分组、加料项、选择数量约束和加价。
- 初始库存、手工入库、手工出库、盘点、订单锁定、释放、支付扣减和退款回补流水。
- B 端商品列表、商品编辑和库存流水页面原型。
- C 端商品列表、详情、规格与加料选择页面原型。

## 3. 不在本阶段实现

- 套餐商品。
- 多门店库存、跨门店调拨和供应商采购。
- 自由装修、营销活动和外链跳转。
- 订单状态机和微信支付真实调用，这些在后续阶段实现。

## 4. 领域规则

- 金额统一使用整数“分”，禁止使用浮点数。
- SKU 规格组合在同一商品内必须唯一，规格展示文本用于订单快照。
- 商品、SKU、加料项上架状态由服务端校验，客户端展示不能作为安全边界。
- 支付未验证时允许保存付费商品草稿，禁止正式上架付费商品。
- 所有库存变化必须写入库存流水，流水包含租户、门店、SKU、变化类型、数量、业务单号和原因。
- 锁定库存只减少可售库存并增加锁定库存；支付成功确认扣减锁定库存；取消或超时释放锁定库存；退款回补可售库存。
- 商品和库存操作必须校验资源属于当前服务端可信租户和门店上下文。

## 5. 数据库设计

- `product_category`：商品分类，包含 `tenant_id`、`store_id`、名称、排序和启用状态。
- `product`：商品主表，包含租户、门店、分类、名称、图片、描述、基础价格、状态和排序。
- `product_sku`：SKU 表，包含租户、门店、商品、规格文本、编码、价格、可售库存、锁定库存和预警值。
- `addon_group`：加料分组，包含必选标识、最小和最大选择数量。
- `addon_option`：加料项，包含名称、加价和启用状态。
- `product_addon_relation`：商品与加料分组关联。
- `inventory_record`：库存流水，禁止物理删除。

所有租户业务唯一索引必须包含 `tenant_id`，门店相关数据同时包含 `store_id`。
