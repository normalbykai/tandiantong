# 04 商品、SKU、加料与库存实施计划

## 1. 功能单元

1. 商品库存数据库迁移：创建分类、商品、SKU、加料和库存流水表。
2. 商品发布领域服务：校验 SKU 组合、价格、加料规则和支付配置状态。
3. 库存领域服务：支持初始库存、手工调整、订单锁定、释放、支付扣减和退款回补。
4. 前端原型：补齐 B 端商品列表、商品编辑、库存流水和 C 端商品展示、规格加料选择。
5. 自动化验证：补充 SKU、加料、库存和租户隔离测试。

## 2. 验证方式

- `mvn -pl tandiantong-catalog -am test`
- `mvn test`
- `npm run typecheck`
- `npm run build`
- `git diff --check`

## 3. 风险控制

- 不引入采购、调拨或套餐抽象。
- 不让客户端提交的 `tenant_id`、`store_id` 参与业务归属判断。
- 库存变化先由应用服务集中维护，后续持久化层只负责原子条件更新和流水落库。
