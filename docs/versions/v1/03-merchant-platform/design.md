# 03 统一后台与商户开通阶段设计

## 1. 阶段目标

本阶段交付平台开通商户与统一后台骨架。平台管理员可以创建商户、初始化单门店、生成首个租户管理员邀请、维护支付配置状态和小程序码入口标识。商户端获得后续商品、订单、预约和核销页面承载框架。

## 2. 范围

- B 端 Vue 3、TypeScript、Vite、Element Plus 工程骨架。
- 统一登录页，区分“平台管理”和“商户管理”身份。
- 平台工作台、商户列表、新建商户向导、商户详情和审核工作台页面。
- 商户工作台、核销中心入口、商品/订单/预约/数据/员工/设置菜单占位。
- 后端商户开通应用服务：创建租户、门店、首个管理员邀请和默认入口码。
- 后端支付配置状态模型：未配置、待验证、已验证、验证失败。
- 小程序码入口使用随机 `scene_key`，禁止直接暴露 `tenant_id`。

## 3. 不做事项

- 不实现商户自主入驻。
- 不实现平台分账、服务商模式结算或真实微信支付配置验证。
- 不实现渠道码、来源统计和营销活动。
- 不实现商品、订单和预约业务详情。

## 4. 后端模型

商户开通聚合包含：

- 租户：名称、编码、状态。
- 门店：租户内默认门店，第一阶段单门店。
- 首个管理员邀请：手机号、邀请码、有效期、状态。
- 商户入口码：随机 `scene_key`、状态和过渡失效时间。
- 支付配置状态：只记录配置状态和脱敏商户号，不保存真实密钥。

## 5. 前端模型

统一后台使用一个前端工程。登录后根据 `domain` 切换平台或商户菜单。桌面端左侧固定菜单，移动端抽屉菜单；移动端优先展示核销入口。

## 6. 验收标准

- 前端可安装依赖、类型检查并构建。
- 页面文案全部为中文。
- 平台商户开通流程在本地模拟数据下可走通。
- 后端商户开通服务可生成租户、门店、管理员邀请和随机入口码。
- 支付未验证时明确阻止后续付费业务上架，真实验证步骤记录到接入说明。
