# 05 购物车、订单、微信支付与退款设计

## 1. 阶段目标

本阶段打通商品购买交易闭环。顾客在当前商户购物车选择商品后创建订单，系统保存商品、SKU 和加料快照，锁定库存，调用微信支付适配；支付成功后确认扣减库存并进入待核销状态；核销前支持整单退款并回补库存。

## 2. 范围

- 当前商户购物车与确认订单模型。
- 商品订单、订单明细快照、支付记录、退款记录和状态日志。
- 创建订单幂等、支付回调幂等、退款幂等。
- 微信支付适配接口和安全本地模拟实现。
- 待支付订单取消和库存释放。
- 核销前整单退款和库存回补。
- B 端订单列表、订单详情和整单退款页面原型。
- C 端购物车、确认订单、支付结果和订单详情页面原型。

## 3. 不在本阶段实现

- 部分退款。
- 核销后在线退款。
- 平台分账和服务商结算。
- 依据前端跳转结果修改支付状态。
- 真实微信支付凭据接入成功声明。

## 4. 交易规则

- 订单主表使用 `sales_order`，避免数据库保留词。
- 创建订单时必须按服务端可信租户和门店上下文重新校验商品、SKU、加料、价格和库存。
- 订单明细必须保存商品名称、SKU 规格、加料名称和成交价格快照。
- 金额使用整数“分”，订单应付金额由服务端计算。
- 创建待支付订单时锁定库存；支付成功确认扣减；取消或超时释放；退款成功回补可售库存。
- 微信支付回调必须验签，并通过平台订单号反查订单和租户。
- 重复创建、重复回调和重复退款必须返回一致业务结果，不重复扣减库存或退款。

## 5. 状态机

商品订单状态：

```text
待支付 -> 待核销 -> 已完成
   |        |
   v        v
已取消   退款中 -> 已退款
```

已完成订单禁止普通接口回退状态或发起在线退款。
