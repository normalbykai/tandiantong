# 05 购物车、订单、微信支付与退款实施计划

## 1. 功能单元

1. 数据库迁移：创建订单、订单明细、支付、退款、状态日志和幂等记录表。
2. 微信支付适配：定义统一接口，提供本地模拟支付和退款实现，真实凭据留给部署环境接入。
3. 订单应用服务：创建订单、锁定库存、生成支付请求、取消订单、处理支付回调。
4. 退款应用服务：核销前整单退款、退款幂等和库存回补。
5. 前端原型：B 端订单列表/详情/退款，C 端购物车/确认订单/支付结果/订单详情。
6. 自动化验证：覆盖状态机、幂等、回调验签失败、重复回调、退款和租户隔离。

## 2. 验证方式

- `mvn -pl tandiantong-order -am test`
- `mvn test`
- `npm audit --audit-level=moderate`
- `npm run typecheck`
- `npm run build`
- `git diff --check`

## 3. 风险控制

- 微信支付真实商户号、证书和密钥禁止提交仓库。
- 支付回调不依赖登录态或客户端租户参数。
- 事务边界放在订单应用服务，真实外部网络调用由适配层控制超时和可重试错误。
