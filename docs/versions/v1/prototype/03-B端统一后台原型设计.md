# 摊点通 V1 B 端统一后台原型设计

## 1. 总体模型

统一后台共用一个登录页面和前端工程。平台账号进入平台权限域，商户账号进入商户权限域。菜单隐藏只用于体验，后端接口必须分别鉴权。

桌面端使用左侧菜单和顶部栏；移动端使用抽屉菜单，优先展示核销、订单和待办。

## 3. B-C-01 统一登录

```text
┌─────────────────────────────────────┐
│ 摊点通                              │
│ 让线下零售与服务经营更简单            │
│                                     │
│ [平台管理] [商户管理]                │
│ 手机号  [请输入手机号              ] │
│ 密码    [请输入密码                ] │
│ [ ] 记住当前身份       [忘记密码]    │
│              [登录]                 │
└─────────────────────────────────────┘
```

失败统一提示“账号或密码错误”。邀请链接自动进入商户管理身份页签。登录成功后返回权限域、用户信息、菜单和操作权限。

## 4. B-P-03 新建商户向导

步骤为“主体信息、门店资料、首个管理员、品牌设置、确认创建”，支持保存草稿。

```text
新建商户
[1 主体信息]—[2 门店资料]—[3 管理员]—[4 品牌]—[5 确认]

主体名称（必填） [春风小铺有限公司           ]
联系人（必填）   [张晓春                    ]
联系电话（必填） [138****8000               ]
证件资料         [上传营业执照]

[保存草稿]                              [下一步]
```

最终确认页展示完整摘要和缺失项。创建成功后状态为“待启用”，发送管理员邀请，并将支付配置列为高优先级待办。

## 5. B-P-05 商户审核工作台

```text
商户审核工作台
[待启用 12] [需修改 4] [待复核 6] [风险复核 2]

筛选：商户名称｜负责人｜提交日期｜风险等级  [查询]

春风小铺   待复核   提交于 7月12日
管理员已激活｜品牌已配置｜支付待配置｜商品 3 项
[查看详情] [开始审核]
```

审核详情检查主体、门店、管理员、品牌、商品或免费服务、支付状态和小程序码。驳回必须勾选结构化问题并填写说明。高风险商户转交超级管理员，禁止自审。

## 6. B-P-04 商户详情

顶部展示商户名称、租户状态、营业状态、支付状态和管理员状态。页签包括“基本资料、门店与品牌、管理员、支付配置、小程序码、审核记录、操作日志”。

平台可执行提交审核、要求修改、启用、停用、重新邀请管理员和查看支付验证结果。商户展示资料由商户管理员维护，平台敏感资料只允许平台角色修改。

## 8. B-T-03 商品分步编辑

步骤为“基础信息、规格与 SKU、加料配置、库存与上架、预览保存”。

- 基础信息：名称、分类、图片、描述。
- 规格与 SKU：规格维度、规格值、组合、价格、编码。
- 加料配置：分组、必选或可选、最少和最多数量、加价。
- 库存与上架：初始库存、预警值、排序、上架状态。
- 预览保存：C 端双列和点餐布局预览、配置问题检查。

删除已使用规格值时显示受影响 SKU。支付未验证时允许保存草稿，但禁止正式上架付费商品。

## 9. B-T-05 与 B-T-06 订单

桌面订单列表包含状态页签、筛选区和表格。移动端转为卡片，突出取餐号、预计取餐时间、金额和核销状态。

订单详情顶部展示状态、取餐号、金额、时间和操作。下方页签为“订单信息、支付退款、核销记录、操作日志”。手机号默认脱敏；退款与核销必须二次确认。

## 10. B-T-09 预约日历

桌面端支持月历和周历，显示每天预约数、剩余容量、已满和暂停时段。点击日期展开时段，点击时段查看预约列表。移动端改为按日期分组的日程列表。

## 11. B-T-10 移动核销中心

```text
核销中心
[ 扫描顾客二维码 ]

取餐号或手机号后四位
[ A023 / 8000             ] [查询]

待核销
A023  桂花拿铁×1、三明治×1
预计取餐 15:30                  [查看]

预约：咖啡体验课
今天 16:00  张女士               [查看]
```

核销详情必须展示商品或服务摘要、时间和顾客脱敏信息，并要求“确认核销”。重复核销显示原核销时间和操作人，不再显示可执行按钮。

## 12. B-T-12 经营数据

包含交易概览、商品分析、SKU 与加料分析、预约分析。支持日期筛选和按当前条件导出。导出按钮单独鉴权，大数据量显示后台生成进度和文件有效期。

## 13. B-T-13 与 B-T-14 员工和角色

员工账号支持邀请、停用、重新发送邀请、角色分配和查看最近登录。角色管理支持名称、说明和权限树；租户管理员不能授予本人没有的权限。

## 14. 响应式与异常状态

- 桌面表格在移动端变为卡片，不横向压缩关键字段。
- 无权限页面显示明确说明并提供返回工作台入口。
- 空数据页面根据权限提供“新建商品”“邀请员工”等动作。
- 支付配置、审核和导出失败必须保留已填写内容并允许重试。
- 租户停用后商户账号不能登录；暂停营业仍可处理已有订单和核销。
