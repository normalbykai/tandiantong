# 权限与平台系统管理开发总结

## 一、今日完成内容

### 1. 平台权限管理

- 完成平台账号、平台角色、平台权限的后台管理接口和页面联动。
- 登录响应增加当前用户的权限编码集合 `permissionCodes`。
- 后端 API 继续通过 `@SaCheckPermission` 执行接口级鉴权。
- 前端菜单、路由和按钮统一复用业务权限编码，通过 `hasPermission` 控制展示。
- 权限编码以业务操作为中心，不再单独维护按钮权限编码。
- 角色权限配置页面仅展示业务权限，不再拆分独立按钮权限。

### 2. 权限授权规则调整

- 移除首席管理员和系统管理员的自动补权逻辑。
- 移除基于 `is_authority_role` 的平台权限绕过逻辑。
- 平台角色是否能够执行操作，完全取决于角色实际分配的权限编码。
- 拥有角色权限配置权限的角色，可以按该权限配置平台角色权限，不再额外限制只能授予操作者已有权限。
- 保留权限域、角色状态和权限点归属校验，避免配置不存在或跨权限域的权限。
- 权限变更继续记录平台操作审计。

### 3. 平台系统管理

- 新增平台系统配置管理，包括品牌信息和账号安全策略。
- 新增平台通用字典管理，包括字典项新增、编辑和启停。
- 新增账号重置密码策略，支持随机临时密码和固定临时密码模式。
- 固定临时密码仅保存 BCrypt 哈希，不保存明文。
- 增加对应 Flyway 迁移和平台权限编码。

### 4. 管理端菜单和页面

- 平台侧边栏调整为：

  ```text
  系统
  ├─ 操作日志
  ├─ 系统管理
  └─ 权限管理
     ├─ 平台账号
     ├─ 平台角色
     └─ 平台权限
  ```

- 权限管理支持展开和收起，默认展开。
- 角色权限弹窗默认收起全部业务分组。
- 权限弹窗按权限编码分为“查看权限”和“业务操作权限”。
- `system`、`dictionary` 和系统安全策略相关权限统一归入“系统管理”分组。

### 5. 后台基础能力

- 完善统一错误响应中的权限提示和追踪号展示。
- 完善平台商户列表筛选和分页展示。
- 完善后台消息提示、公共列表布局和数据表格组件。
- 完善登录会话、角色展示和权限缓存处理。

## 二、权限编码约定

权限编码使用稳定的业务语义，不直接绑定前端按钮或具体 URL：

| 权限编码示例 | 含义 |
| --- | --- |
| `platform:account:read` | 查看平台账号 |
| `platform:account:create` | 新增平台账号 |
| `platform:role:permission:assign` | 配置平台角色权限 |
| `platform:system:config:read` | 查看平台系统配置 |
| `platform:system:config:update` | 修改平台系统配置 |
| `platform:dictionary:update` | 编辑平台字典项 |

以 `:read` 结尾的权限在前端权限配置中归入“查看权限”，其他权限归入“业务操作权限”。

## 四、使用注意事项

- 权限或角色调整后，建议退出并重新登录，使登录响应中的权限编码集合刷新。
- 菜单和按钮隐藏只用于改善操作体验，后端 API 鉴权仍是最终安全边界。
- 当前权限管理范围主要覆盖平台权限域，商户权限域仍按后续阶段设计逐步接入。
